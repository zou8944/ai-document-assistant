# 聊天斜杠命令支持

## 概述

在聊天输入框中支持斜杠命令，用户输入 `/` 时弹出命令菜单，首个命令 `/clear` 用于清空当前会话的全部对话历史。

## 范围

- 仅 `/clear` 一个命令，但架构支持后续扩展
- `/clear` 永久删除后端该会话的所有消息，同时清空前端显示

## 后端

### 新增端点

`DELETE /api/v1/chats/{chat_id}/messages`

- 调用已有的 `ChatMessageRepository.delete_by_chat(chat_id)`
- 同时重置 `Chat.message_count = 0`，`Chat.last_message_at = null`
- 响应：`{ code: 0, data: { deleted_count: N } }`

### 新增服务方法

`ChatService.clear_chat_messages(chat_id: str) -> int` — 删除消息并重置 chat 统计字段，返回删除数量。

## 前端

### API 客户端

`apiClient.clearChatMessages(chatId: string)` — 调用新 DELETE 端点。

### useChat Hook

新增 `clearMessages()` 方法：调用 API → 设置 `messages = []` → 重置分页状态 (`hasMoreOlder = false`, `isLoadingOlder = false`)。

### SlashCommandSelector 组件

新文件：`frontend/src/components/chat/SlashCommandSelector.tsx`

- 当输入框内容以 `/` 开头时显示在输入框上方
- 显示命令列表：命令名 + 描述
- 支持键盘导航（↑↓ Enter Escape）
- 输入文本自动过滤匹配命令
- 点击外部或 Escape 关闭

### RichTextInput 改动

- 追踪 `showSlashMenu` 状态（输入以 `/` 开头时激活）
- 根据 `/` 后的文本过滤命令列表
- 当菜单可见时，键盘事件转发给 SlashCommandSelector
- 选中命令时通过 `onSlashCommand(commandId)` 回调通知父组件

### ChatInterface 改动

- 定义命令列表：`[{ id: 'clear', label: '/clear', description: '清空当前会话的所有对话历史' }]`
- 处理 `onSlashCommand`：若为 `clear` 则调用 `clearMessages()` 并清空输入框

## 交互流程

```
用户输入 "/" → 弹出命令菜单显示 /clear
  ├─ 方向键导航，Enter 选中
  ├─ 输入 "/cle" → 过滤到 /clear
  ├─ Enter 选中 /clear → 清空输入框，调用 clearMessages()
  ├─ Escape → 关闭菜单，保留输入
  └─ 输入不匹配的文本 → 菜单自动关闭
```
