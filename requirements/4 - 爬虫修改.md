# 爬虫修改需求 - URL 输入与知识库构建

## 原始需求

### 爬取范围
- 需要遍历所有 a 标签的地址
- 所有在当前域名下的页面全都要爬取

### 爬取结果呈现
- 以原网页的形式呈现，超链接依旧是超链接，只不过现在链接的是下载下来的本地文件
- 每个文件存储两种格式：一是转换后的 Markdown 文件、另一个是原 HTML 静态文件
- 页面还需要存储原页面的 URL
- 所有页面下载完成后，需要使用 AI 根据页面的 URL 和页面标题做逻辑处理，给出一个 site-map，使页面不是独立存在，而是体现出联系
  - 是否需要用 canvas 画一下这些文档之间的关系？

---

## 需求评估

### 1. 全站爬取（遍历所有 a 标签 / 同域名所有页面）

**结论：已基本实现，有细节缺口**

`crawl_recursive` 已具备 BFS 遍历、同域名过滤、`recursive_prefix` 限制能力。现有缺口：

- **动态渲染页面**：目前用 `requests + BeautifulSoup`，遇到 SPA（React/Vue 文档站）会拿到空内容。**已决策：暂不支持动态页面，不引入 Playwright**
- **sitemap.xml 优先策略缺失**：大多数文档站都有 `/sitemap.xml`，先解析它比 BFS 更快更全（**已决策：优先使用 sitemap.xml，不存在时降级为 BFS 遍历 a 标签**）
- **去重不完整**：带 query 参数的 URL、trailing slash、大小写不一致会导致同一页面爬多次
- **中断续爬缺失**：大站爬到一半中断，目前只能从头重来

### 2. 双格式存储（HTML + Markdown）+ 记录原始 URL

**结论：可行，改动量小**

当前 `_fetch_page_with_cache` 只存 HTML，`markdownify` 转换已有。需改动：
- 存储时同时写 `.html` 和 `.md` 两个文件
- Markdown 文件头加 frontmatter 记录元信息：

```markdown
---
source_url: https://docs.example.com/guide/intro
title: Introduction
crawled_at: 2026-04-08T10:00:00
---
```

### 3. 本地链接重写（HTML 中链接指向本地文件）

**结论：可行，但复杂度较高，是"离线浏览体验"的核心**

现有代码中**完全缺失**，需新增一个爬取后的后处理步骤：
- 爬取完成后遍历所有 HTML 文件
- 将已爬取页面的 URL 替换为本地相对路径（如 `../guide/intro.html`）
- 未爬取的链接保持原 URL 不变

**已决策**：静态资源（CSS / 图片 / JS）保留对原始 URL 的引用，不下载到本地。HTML 使用浏览器组件渲染，可直接加载远端资源，保证页面样式正常显示。

### 4. AI 生成 Sitemap（建立页面层级与联系）

**结论：可行，应分两步做，不要全靠 AI**

| 方法 | 优点 | 缺点 |
|------|------|------|
| 纯 AI 推断 | 语义理解强 | 不稳定、成本高 |
| 纯 URL 路径分析 | 快速稳定 | 无法处理 URL 与内容不对应的情况 |
| **两者结合（推荐）** | 准确 + 有语义 | 实现稍复杂 |

推荐：先用 URL 路径自动构建树结构，再用 AI 为每个节点补充简短描述 / 分类标签，不让 AI 决定层级关系。

### 5. Canvas 可视化文档关系

**结论：可行，建议分两阶段**

- **Phase 1**：树形列表（折叠展开），成本低，使用现有前端组件即可落地
- **Phase 2**：图形化视图（`react-flow` 或 `D3.js`），展示页面间链接关系网络，而不仅是层级树

**已决策**：Canvas 可视化为必选 MVP 功能，两个阶段均需实现。

## 数据库变更方案

### 变更汇总

| 表 | 字段 | 类型 | 说明 |
|---|---|---|---|
| `documents` | `source_path` | `TEXT NULL` | 爬取 URL 的路径部分（如 `/guide/intro`），用于链接重写目标 `docs/{collection_id}/{source_path}`；本地文件为 NULL |
| `documents` | `html_content` | `TEXT NULL` | 链接重写后的 HTML，供前端离线预览；`content` 字段继续存 Markdown 用于 RAG；本地文件为 NULL |
| `collections` | `sitemap_json` | `TEXT NULL` | AI 生成的站点树结构 JSON（Phase 3 产出） |

### 设计说明

- `source_path` 而非 `doc_name`：强调它是原始路由路径，与 `uri`（完整 URL）形成对应，与 `name`（展示用标题）区分
- `html_content` 独立存储而非复用 `content`：RAG 管道直接消费 `content`（Markdown），混用会导致所有 RAG 代码需要按 `mime_type` 分叉
- `mime_type` 字段保持不变：对本地文件（PDF、DOCX 等）仍有实际用途（驱动处理管道、下载响应头）

## 实现方案

### 总体规划（4 个阶段）

```
Phase 1 - 存储格式升级
  ├── 同时保存 .html 和 .md 两种格式
  ├── Markdown frontmatter 记录 source_url / title / crawled_at
  └── 爬取结果写入数据库 document 表，与现有 collection 体系打通

Phase 2 - 链接重写（离线预览）
  ├── 爬取完成后的后处理步骤
  ├── 替换已爬取页面的 href 为本地相对路径
  └── 静态资源策略：初版跳过，仅处理页面链接

Phase 3 - AI Sitemap 生成
  ├── 用 URL 路径层级自动构建基础树结构
  ├── LLM 调用为每个节点补充描述与分类
  └── 写入数据库，暴露 API 给前端

Phase 4 - 可视化
  ├── 树形列表（折叠展开），快速落地
  └── react-flow 图形视图（可选升级）
```

### 优先级建议

**Phase 1 → Phase 3 → Phase 2 → Phase 4**

- Phase 3（Sitemap）对 RAG 检索的上下文理解很有帮助，应优先于 Phase 2
- Phase 2（链接重写）是纯 UX 功能，对核心问答能力没有影响，可后做
- Phase 4（可视化）依赖 Phase 3 的数据，排最后

### Phase 1 详细设计

**文件存储结构**
```
cache/
└── docs.example.com/
    ├── index.html          # 原始 HTML
    ├── index.md            # 转换后 Markdown（含 frontmatter）
    ├── guide/
    │   ├── intro.html
    │   └── intro.md
    └── api/
        ├── reference.html
        └── reference.md
```

**改动点**：`simple_web_crawler.py` 的 `_fetch_page_with_cache` 方法，存 HTML 的同时多写一个 `.md` 文件。

### Phase 2 详细设计

**后处理流程**
1. 收集所有已爬取 URL → 本地路径的映射表
2. 遍历所有 `.html` 文件，用 BeautifulSoup 解析
3. 对每个 `<a href>` 检查是否在映射表中，是则替换为相对路径
4. 写回文件

**静态资源策略（已决策）**：所有静态资源（CSS / 图片 / JS）保留原始绝对 URL，不做修改、不下载。HTML 通过浏览器组件渲染，组件可直接访问网络加载远端资源，页面样式与线上一致。只处理 `<a href>` 的页面链接重写。

### Phase 3 详细设计

**两步生成 Sitemap**

Step 1 - URL 路径解析（无需 LLM）：
```
https://docs.example.com/guide/intro
  → { level: ["guide", "intro"], title: "Introduction" }
```

Step 2 - LLM 语义增强：
- 输入：URL 路径 + 页面标题列表
- 输出：每个节点的简短描述、所属分类标签
- Prompt 限制输出 JSON 格式，避免幻觉

**数据库存储**：在 `collections` 表新增 `sitemap_json` 字段，存储整棵树的 JSON。

### Phase 4 详细设计

**树形列表（Phase 4a）**：
- 使用 React 递归组件渲染
- 点击节点展开/折叠，点击叶节点跳转到对应文档

**图形视图（Phase 4b）**：
- 使用 `react-flow` 渲染节点关系图
- 节点 = 页面，边 = 链接关系
- 支持缩放、拖拽、点击跳转
