# 知识库页面修改需求

## 原始需求

目前这个知识库功能并不好用，需要做如下修改：

**第一，结构上要收敛。**

不要再分两个 tab，也不要有一个独立的"站点结构页"。所有能力统一到一个知识库页面里：左边是分类导航，右边是内容区，上面是搜索。结构页这种"需要专门进去看结构"的设计要彻底去掉，因为用户不应该先想结构，而是直接开始找内容。

**第二，入口要清晰。**

默认页面不再是文档列表，而是一个 AI 生成的 README 首页，用来告诉用户"你想找什么应该去哪"。它不是普通文档，而是一个导航面板，点击里面的项，本质上是在帮用户做筛选/跳转，而不是跳转到某一篇具体文档。

README 是根据所有标题 Title、URL 和根文档一起生成的；点击 README 对应的链接会跳转到对应的文档阅读页面。

**第三，查找路径要顺畅。**

用户找内容的方式应该只有三种，而且这三种要打通：

- 搜索（最直接的方式）
- 左侧分类（缩小范围）
- 浏览列表（做选择）

导航树不再是展示所有页面，而是用来做"分类过滤"；文档列表不再按 URL 排，而是围绕"可读标题 + 命中内容"；点击文档后也不要弹窗，而是在右侧直接进入阅读状态，让用户始终在同一个上下文里操作。

导航树和列表是需要联动的，即如果在文档列表中选中某个列表，则导航树要调到对应位置。

## 设计决策w

**去掉 sitemap_json，一步到位生成 README。**

原方案是先生成 sitemap_json（结构化 JSON），再基于 sitemap 生成 README。经讨论，sitemap 本身不再有前端消费者（SitemapView 组件将被删除），作为中间产物没有独立价值。改为：爬取完成后直接调用 LLM 一次，从文档列表生成 README + 分类数据，省掉 sitemap 这个中间层。

## 实现方案

### 整体布局

```
+--------------------------------------------------+
|  Header: 返回按钮 + 知识库名 + 导入按钮(可折叠)    |
+--------------------------------------------------+
|  [搜索框: 搜索文档...]                              |
+--------------------------------------------------+
| 左侧导航栏(240px) |  右侧内容区                     |
|                   |                               |
|  - 全部文档         |  默认：AI 生成的 README 首页   |
|  - 分类 A (3)       |  选中分类：文档列表             |
|  - 分类 B (5)       |  选中文档：文档阅读面板         |
|  ...                |                               |
+--------------------------------------------------+
```

### Part 1: 后端 — 新增 `readme_content` + `categories_json` 字段

#### 1.1 数据库：新增两列，删除一列

- `backend/database/models/collection.py`:
  - 添加 `readme_content: Mapped[str | None] = mapped_column(Text, nullable=True)` — AI 生成的 Markdown README
  - 添加 `categories_json: Mapped[str | None] = mapped_column(Text, nullable=True)` — 分类数据 JSON
  - 删除 `sitemap_json` 字段
- `backend/models/dto.py` — `CollectionDTO` 添加 `readme_content`、`categories_json`；删除 `sitemap_json`
- 生成 Alembic migration：`alembic revision --autogenerate -m "replace sitemap_json with readme_content and categories_json"`
- 运行 `alembic upgrade head`

#### 1.2 LLM：新增 `generate_readme()` 方法

- `backend/services/llm_service.py` — 添加 `generate_readme(pages: list[dict]) -> str`
- 输入：`[{"path": "...", "title": "..."}]`（所有 crawled 文档的 path 和 title）
- Prompt 要求 LLM 同时输出两部分：
  1. README Markdown 正文（自然语言指南，按分类组织，文档名用 `[标题](doc://{path})` 链接）
  2. 末尾附带一个 JSON 代码块 `<!-- categories -->` 包含分类数据
- 或者更简洁的方案：Prompt 分两步 — 先生成分类，再生成 README，但为了减少 LLM 调用次数，用单次调用 + 结构化 prompt：
  - 要求 LLM 返回 JSON：`{"readme": "markdown content...", "categories": [{"category": "...", "pages": [{"path": "...", "title": "...", "description": "..."}]}]}`

#### 1.3 任务流：替换 sitemap 生成为 readme 生成

- `backend/services/task_service.py`:
  - 将 `_generate_sitemap()` 替换为 `_generate_readme()`
  - `_generate_readme()` 流程：
    1. 获取所有 crawled 文档
    2. 构建 `pages = [{"path": d.source_path, "title": d.name or ""} for d in crawled]`
    3. 调用 `llm_service.generate_readme(pages)` 得到 JSON 字符串
    4. 解析 JSON，分别提取 `readme` 和 `categories`
    5. 调用 `collection_service.update_readme(collection_id, readme_content, categories_json)`

#### 1.4 API：新增端点

- `backend/services/collection_service.py` — 添加 `update_readme()` 和 `get_readme()` 方法
- `backend/api/routes/collections.py` — 新增 `GET /collections/{id}/readme` 端点，返回 `{readme_content, categories_json}`
- `backend/models/responses.py` — 添加 `ReadmeResponse` 模型

#### 1.5 清理旧的 sitemap 相关代码

- `backend/services/llm_service.py` — 删除 `generate_sitemap()` 方法
- `backend/services/task_service.py` — 删除 `_generate_sitemap()` 方法
- `backend/services/collection_service.py` — 删除 `update_sitemap()` 方法
- `backend/api/routes/collections.py` — 删除 `GET /collections/{id}/sitemap` 端点
- `backend/models/responses.py` — 删除 `SitemapResponse` 模型
- `backend/database/models/collection.py` — 删除 `sitemap_json` 字段（或保留但不再使用，避免 migration 麻烦）
- `backend/models/dto.py` — 删除 `sitemap_json` 字段（或保留）

> 注意：`sitemap_json` 字段可以从数据库中删除（通过 migration），也可以保留不使用。建议通过 migration 删除以保持整洁。

### Part 2: 前端 — 工具函数

**新建 `frontend/src/utils/categoryParser.ts`：**
- 类型定义：
  ```typescript
  interface SitemapPage { path: string; title: string; description?: string }
  interface CategoryGroup { category: string; pages: SitemapPage[] }
  ```
- `parseCategories(json: string | null): CategoryGroup[]` — 解析 categories_json
- `getCategoryNames(groups): string[]` — 提取分类名列表
- `getPageCountByCategory(groups): Map<string, number>` — 各分类文档数量

### Part 3: 前端 — 组件重构

#### 3.1 `SidebarNav.tsx`（新建）— 左侧分类导航栏

- Props: `categories: CategoryGroup[], activeCategory: string | null, onCategorySelect, highlightedCategory`
- "全部文档"作为第一项
- 各分类显示名称 + 文档数量 badge
- 选中项高亮（`bg-blue-50 text-blue-600`）
- `highlightedCategory` 用于文档选中时联动高亮对应分类

#### 3.2 `ReadmePanel.tsx`（新建）— AI 生成的 README 首页

- Props: `readmeContent: string, onDocClick: (path: string) => void`
- 渲染 README markdown 内容（使用简单的 markdown → HTML 转换）
- 拦截 `doc://` 链接点击：解析 path，查找对应文档，调用 `onDocClick`
- 如果没有 readme_content（纯文件知识库或旧数据），fallback 显示文档列表
- 样式：prose 排版，`max-w-none`

#### 3.3 `DocReader.tsx`（新建）— 文档阅读面板

- Props: `doc, previewUrl, onBack`
- 顶部 bar：返回按钮 + 文档标题 + 来源外链（新窗口打开）
- 主体：iframe 加载 `previewUrl`（复用 `getDocumentPreviewUrl`）
- 占满右侧内容区

#### 3.4 `KnowledgeBaseManagement.tsx`（重写）— 三栏布局

**新增状态：**
- `activeCategory: string | null` — 当前选中分类
- `selectedDocId: string | null` — 当前选中文档
- `searchQuery: string` — 搜索关键词
- `readmeContent: string | null` — AI 生成的 README
- `categoriesJson: string | null` — 分类数据
- `importCollapsed: boolean` — 导入区域是否收起

**删除状态：**
- `activeTab`, `sitemapJson`, `loadingSitemap`, `previewOpen`, `previewUrl`, `previewTitle`, `previewSourceUrl`

**右侧内容区条件渲染：**
1. `selectedDocId === null && activeCategory === null && searchQuery === ''` → ReadmePanel
2. `selectedDocId === null` → 文档列表（按 category 或 search 过滤）
3. `selectedDocId !== null` → DocReader

**搜索逻辑：**
- 输入时调用后端 `listDocuments(collectionId, 1, 50, searchQuery)`
- 有搜索词时切换到文档列表视图
- 清空时回到 README 首页

**导入区域：** 可折叠面板，默认收起。

### Part 4: 前端 API 变更

- `frontend/src/services/apiClient.ts`:
  - 删除 `getSitemap()` 方法
  - 新增 `getReadme(collectionId)` 方法，返回 `APIResponse<{ readme_content: string | null, categories_json: string | null }>`
  - `Collection` 接口删除 `sitemap_json`，添加 `readme_content?` 和 `categories_json?`

### Part 5: 清理

- 删除 `frontend/src/components/knowledge/SitemapView.tsx`
- 删除 `frontend/src/components/knowledge/PagePreview.tsx`

## 实现顺序

1. 后端 DB migration + model/dto 更新（添加新字段，删除 sitemap_json）
2. 后端 LLM `generate_readme()` + task_service 集成（替换 sitemap）
3. 后端 API 端点（新增 readme 端点，删除 sitemap 端点）
4. 前端 `utils/categoryParser.ts`
5. 前端 `apiClient.ts` 类型和方法更新
6. 前端 `SidebarNav.tsx`
7. 前端 `ReadmePanel.tsx`
8. 前端 `DocReader.tsx`
9. 前端 `KnowledgeBaseManagement.tsx` 重写
10. 清理旧组件

## 验证方法

1. 运行 migration，启动后端，导入一个网站 URL，等待爬取完成
2. 检查数据库 `collections.readme_content` 和 `categories_json` 字段
3. curl 测试 `GET /collections/{id}/readme` 返回正确数据
4. 启动前端，进入知识库：
   - 左侧导航栏显示分类列表
   - 默认显示 README 导航面板（自然语言内容 + 链接）
   - 点击 README 中的链接 → 进入文档阅读面板
   - 点击左侧分类 → 切换到文档列表视图
   - 搜索框输入关键词 → 切换到列表视图并过滤
   - 点击文档行 → 右侧展开阅读面板
   - 点击返回 → 回到上一级
   - 导航树联动高亮正确
5. 纯文件知识库（无 readme）fallback 到文档列表
6. 导入功能正常（可折叠面板展开后）
